## 图 1 · 对话主页（标准问答）

![Chat Page](./01_chat_page.svg)

**场景**：用户咨询订单物流，AI 通过 RAG 检索给出带引用的回答。

**标注说明**：
| 编号 | 模块 | 设计要点 |
|---|---|---|
| ① | 左侧会话列表 | 按"今天 / 昨天 / 上周"分组；置顶高亮当前会话 |
| ② | 用户气泡 | 深色背景靠右，时间戳在外侧，符合 IM 直觉 |
| ③ | 引用卡片 | 强制每条答案配 1-3 张，标号【1】【2】与正文对应 |
| ④ | 反馈按钮 | 👍/👎/重试/复制 四件套，反馈数据回流评测平台 |
| ⑤ | 推荐 Quick Replies | 基于答案内容生成，引导下一步操作 |
| ⑥ | 输入框 | 支持文本/语音/附件；显示 AI 免责声明 |

## 图 2 · 情绪安抚特殊场景

![Emotion Handling](./02_emotion_handling.svg)

**场景**：用户因质量问题情绪激动，系统检测到负面情绪后切换"情绪关怀模式"。

**标注说明**：
| 编号 | 模块 | 设计要点 |
|---|---|---|
| ① | 情绪识别标签 | 实时显示情绪类型+强度（0-10），可被运营追踪 |
| ② | 共情开场 | 黄色温暖色调横幅，先安抚再解决 |
| ③ | 分步方案 | 推荐方案高亮、备选方案弱化，降低用户决策成本 |
| ④ | 行动按钮 | 主操作蓝色填充、转人工金色提示，永远保留"人工"出口 |
| ⑤ | 右侧监控面板 | SLA 计时、处理路径、智能建议，供管理员/PM 看运营 |

---

## 设计原则

1. **永远给用户出口**：不管 AI 多智能，转人工按钮必须可见。
2. **引用即信任**：每个回答都要有出处，没有就明确告知"未找到"。
3. **情绪先行**：识别到负面情绪时，先共情、再解决、最后提供升级通道。
4. **数据可监控**：右侧面板对 PM/运营开放，方便日常治理与复盘。

## 如何修改

SVG 是矢量格式，用以下任一方式编辑：

- **VSCode** + 插件 `Svg Preview`：边改代码边看效果
- **Figma**：File → Import → 选 SVG，即可拖拽编辑
- **Inkscape**（免费）：开源 SVG 编辑器，类似 Illustrator
- **浏览器**：双击打开看高清版，截图存档
